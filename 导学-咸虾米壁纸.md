## 6. 项目技术定位

**前端（跨端）**。依据：项目基于 UniApp 框架，使用 Vue3 语法，同时编译到微信小程序、H5、App 等多端，核心难点集中在布局适配、性能优化（图片预加载）、多端 API 差异处理与分享链路。

## 7. 核心原理解析

### 7.1 自定义导航栏高度计算

- **问题**：微信小程序原生导航栏被胶囊按钮占据，不同机型状态栏高度不同；H5 和 App 没有胶囊按钮。
- **机制**：通过 `uni.getSystemInfoSync` 获取 `statusBarHeight`，通过 `uni.getMenuButtonBoundingClientRect` 获取胶囊按钮位置，计算 `titleBarPadding` 与 `titleBarHeight`。
- **在本项目中的落点**：`utils/system.js` 与 `utils/layout.js` 提供了完整计算函数；`custom-nav-bar.vue` 使用 `:style` 动态绑定高度，并在下方放置等高的 `fill` 占位块，防止内容被导航栏遮挡。

### 7.2 图片预加载（前后三张）

- **问题**：壁纸预览使用 `swiper` 全屏滑动，若一次性加载全部高清图，内存与网络开销极大。
- **机制**：维护一个 `readImgs` 数组，仅在 `swiperChange` 时计算「当前索引 ±1」三张图片的索引，并去重追加到数组中；模板中用 `v-if="readImgs.includes(index)"` 控制渲染。
- **在本项目中的落点**：`pages/preview/preview.vue` 的 `readImgsFun` 函数。

### 7.3 本地缓存作为页面间数据通道

- **问题**：图片列表数据量大，无法通过 URL `query` 传递；且小程序页面栈跳转后需要保持列表状态。
- **机制**：列表页（`classlist` / `search`）将数据 `uni.setStorageSync('storgClassList', classList.value)`；预览页 `onLoad` 中读取并还原；页面卸载时 `uni.removeStorageSync` 清理。
- **在本项目中的落点**：`pages/classlist/classlist.vue`、`pages/search/search.vue`、`pages/preview/preview.vue`。

### 7.4 条件编译处理多端差异

- **问题**：抖音小程序不支持 `open-type="contact"`；H5 无法调用 `uni.saveImageToPhotosAlbum`；部分页面在抖音小程序需要隐藏自定义导航栏避免冲突。
- **机制**：UniApp 提供的条件编译注释，在编译期保留/删除代码块。
- **在本项目中的落点**：`#ifndef MP-TOUTIAO` 控制自定义导航栏显隐；`#ifdef H5` 提示长按保存；`#ifdef MP` / `#ifndef MP` 切换客服按钮与拨打电话。

### 7.5 请求统一封装与错误码处理

- **问题**：分散的 `uni.request` 调用难以统一加 header、处理异常、做提示。
- **机制**：封装 `request` 函数返回 Promise，统一拼接 `BASE_URL`、注入 `access-key` header；按 `errCode` 分流：`0` resolve，`400` 用 Modal，其他用 Toast。
- **在本项目中的落点**：`utils/request.js`。

### 7.6 广告位动态插入内容流

- **问题**：广告需要在内容列表中按固定位置插入，且只在微信小程序平台展示。
- **机制**：请求到数据后，在 `data.length > 9` 时 `splice(9, 0, { ad: true })`；模板中通过 `v-if="item.ad"` 渲染广告组件，并用 `grid-column: 1/4` 实现网格通栏。
- **在本项目中的落点**：`pages_quick/news.vue`。

## 8. 关键设计决策

| 决策点 | 备选方案 | 最终选择 | 取舍与风险 |
|--------|----------|----------|------------|
| 页面间传数据：Storage vs URL query | URL query 传 id 再二次请求 | **Storage 传完整列表** | 减少二次请求，提升预览页打开速度；风险是 Storage 未及时清理可能占用空间，已在 `onUnload` 中清理 |
| 预览页图片加载：全量 vs 前后三张 | 一次性 `v-for` 渲染全部 `image` | **`v-if` 控制前后三张** | 显著降低内存占用；风险是快速滑动时可能出现短暂白屏，已通过 `circular` 与预加载索引兜底 |
| 自定义导航栏：全局组件 vs 页面内写 | 每个页面自己写导航栏 | **抽取 `custom-nav-bar` 组件** | 统一维护，减少重复代码；风险是抖音小程序原生导航栏冲突，已通过 `#ifndef MP-TOUTIAO` 排除 |
| 下载授权失败处理：Toast vs Modal | 仅 Toast 提示失败 | **Modal 引导打开设置页** | 用户路径更长但授权成功率更高；风险是用户仍可能拒绝，已做二次 Toast 兜底 |
| 排行榜日期选择：内置 picker vs uni-datetime-picker | 自行封装日期逻辑 | **使用 `uni-datetime-picker`** | 减少自研成本，样式通过 `:deep()` 调整；风险是组件库版本升级可能导致样式穿透失效 |
